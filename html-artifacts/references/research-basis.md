# Research Basis

This skill treats HTML artifacts as self-contained visual documents generated by an agent: codebase explainers, PR walkthroughs, implementation recaps, data reports, timelines, presentations, and small explorable models. It does not assume Anthropic's hosted Artifact runtime; the default output is an ordinary local HTML file.

## Resulting Boundaries

- Default to a true offline local file because it is portable, inspectable, private, and independent of one provider's artifact runtime.
- Default to document mode because code reviews, system explanations, and implementation recaps are usually consumed asynchronously.
- Require an evidence ledger because polish can make unsupported claims look more credible.
- Require a complete static story because interaction can hide the conclusion and exclude readers.
- Prefer plain HTML/CSS/SVG/JavaScript because a framework is packaging overhead unless the artifact has real state complexity.
- Prohibit generic visual chrome because it obscures information hierarchy and creates the “complete slop” failure that motivated the skill.
- Test the bundled output at desktop and mobile sizes with keyboard, reduced-motion, accessibility, runtime, and offline checks.
